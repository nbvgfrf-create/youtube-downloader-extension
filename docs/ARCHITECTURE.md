# Architecture

## Overview

Проект состоит из двух основных частей:

1. **Browser Extension** — пользовательский интерфейс в браузере;
2. **Local Python Helper** — локальный backend, который выполняет скачивание, объединение дорожек, хранение настроек и очереди задач.

Связь между ними идёт через локальный HTTP API по loopback-адресу:

```text
http://127.0.0.1:45719
```

## Components

### 1. Browser Extension

#### `content.js`
- встраивает кнопку `Скачать` на страницу YouTube;
- добавляет кнопку `Качество` для выбора формата.

#### `popup.js`
- показывает резервный интерфейс через popup;
- отображает текущие и недавние загрузки;
- позволяет выбрать качество;
- открывает настройки helper.

#### `background.js`
- взаимодействует с локальным helper API;
- отправляет запросы на `127.0.0.1:45719`;
- получает статусы, список форматов и прогресс задач.

### 2. Local Python Helper

#### `app.py`
- запускает локальный API;
- поднимает tray-агент;
- инициализирует сервисы приложения.

#### `http_api.py`
- принимает запросы от расширения;
- отдаёт данные о форматах, загрузках, статусах и настройках;
- реализует локальный HTTP интерфейс.

#### `yt_dlp_service.py`
- работает с `yt-dlp`;
- скачивает видео и аудио;
- подготавливает объединение дорожек через `ffmpeg`.

#### `download_manager.py`
- ведёт очередь задач;
- хранит статусы загрузок;
- управляет прогрессом и завершением задач.

#### `tray.py`
- реализует иконку в системном трее;
- использует Win32 API через `ctypes`;
- позволяет держать helper в фоне.

#### `/settings`
- локальная web-страница;
- используется для управления настройками helper.

## Communication model

Связь extension ↔ helper построена как локальный IPC через HTTP.

### Почему так
- проще отлаживать;
- не нужен прямой запуск `.exe` из расширения;
- логика скачивания остаётся полностью внутри helper;
- расширение остаётся лёгким и отвечает только за UI.

### Канал связи

```text
http://127.0.0.1:45719
```

## Download flow

### Форматы до 360p
Используется единый поток:
- video + audio в одном файле.

### Форматы выше 360p
YouTube часто отдаёт видео и аудио отдельно, поэтому используется схема:

1. скачать `video-only`;
2. скачать `audio-only`;
3. если `audio-only` недоступен, попробовать запасной вариант через `360p`;
4. объединить дорожки через `ffmpeg`;
5. удалить временные файлы.

## Cookies

Поддерживаемые режимы:

- `auto`
- `browser`
- `file`
- `none`

### Поведение
- если YouTube не требует авторизации, download проходит без cookies;
- если YouTube блокирует доступ к потокам, helper возвращает понятную ошибку;
- пользователь может включить cookies через настройки.

### Особенность Яндекс Браузера
Текущая версия `yt-dlp` не умеет напрямую читать cookies из Яндекс Браузера как отдельного браузера.  
Поэтому рекомендуется:
- сначала пробовать `auto`;
- при необходимости использовать `cookies.txt`.

## Packaging and launch

### Dev launch
- `run_helper.cmd`
- `run_helper.ps1`

### Build to executable
- `app/helper/build.ps1`

### Installer
- `installer/setup.iss`

## Browser installation limitation

Полностью автоматическая и универсальная установка локального расширения в Chrome / Edge / Яндекс Браузер без магазина или policy-install не является надёжной.

Поэтому текущая схема такая:
- helper может быть собран в standalone `.exe`;
- extension ставится как unpacked extension в режиме разработки.

## Design goals

- минимальная нагрузка на браузер;
- устойчивый фоновый helper;
- понятные ошибки для пользователя;
- простая сборка и отладка;
- возможность в будущем упаковать helper в installer;
- архитектура без прямой зависимости расширения от внутренней логики скачивания.
